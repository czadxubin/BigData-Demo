### 该demo演示：计算多个订单中每组订单中单个商品最大交易金额统计
如：
订单id:0001,购买了小米手机1999，ipad air2 3368 
那么该笔订单中ipad的交易额最大

#### 用到的知识点
WritableComparable.compareTo()


#### 测试数据
从左到右，依次为：订单ID、商品名称、交易金额,分隔符：","
0001,小米音箱,99
0001,小米电视,4999
0002,iphone7,5388
0002,mac book,12888
0003,华为荣耀8,2388
0002,ipad air2,3388
0003,华为平板,3288
0001,小米2,1299

#### 思路分析(方案一)
最简单的方式，就是利用订单号作为Mapper的输出key,这样同一个订单的数据进入到同一个reducer，在从reducer中找到最大值即可。
但是此方案效率较低，需要遍历所有的values

#### 思路分析(方案二：本次使用)
1.将每条记录作为一个整体javaBean同时作为一个Mapper的输出key,value输出null(不关心value)
2.Mapper输出到Reducer时,首先按照订单号分组，key值就是我们要的结果
3.这样在Reducer只需输出结果

这个方案的重点：
1.我们要确保同一个订单输出到同一个reducer(除非你只有一个reducer)，这里就需要我们按照订单号进行Partioner了
2.排序问题：mapper阶段我们的输出结果排序两次，先按照订单号排序，再按照单商品的交易金额排序。这样才能确保reducer阶段的第一个key就是结果
